# Performance evidence registry

`baseline.json` is the checked-in archival performance evidence for every production AdapterCapability. It was generated by `tools/benchmark_capsules.py` on the environment recorded inside the report.

Consumers must match records by `capability_id`; array position is not an identity. A manifest evidence URI has the form:

```text
registry/performance/baseline.json#capability:<capsule>/<strategy>/<backend>
```

`tools/sync_edge_weights.py` materializes each measured capability into the owning standalone Capsule's root `edge-weight.json`. `registry/support-matrix.json` points directly to those files. CI rejects a missing, unexpected or stale local weight file.

Use the raw cost vector for graph planning. `load_0_to_100 = 100 - score.overall_0_to_100` is a calibrated higher-is-more-expensive summary for same-profile ranking and must not override semantic, loss, security or resource constraints. See `docs/13-performance-evidence.md`.
