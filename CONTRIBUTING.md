# Contributing

EverythingX 计划长期接受社区讨论、问题报告和设计建议，但著作权链必须保持清晰，以便项目能够合法授予商业许可证。

在仓库发布正式的贡献者许可协议（CLA）并建立签署流程前：

- 欢迎提交 Issue、错误复现、格式资料链接和不包含第三方受保护代码的建议；
- 请不要提交大段代码、完整实现或受第三方许可证约束的材料；
- 维护者不会合并会产生独立著作权贡献的 Pull Request；
- “提交了 PR”不表示贡献者转让著作权，也不表示项目获得商业再许可权。

未来 CLA 至少需要覆盖：贡献者权属声明、著作权许可、商业再许可、专利许可、第三方材料披露及不可撤销性。在该流程上线前，核心代码由著作权人或与其另有书面协议的开发者维护。
